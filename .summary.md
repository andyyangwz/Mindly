## Goal
- Update Home Page analytics to use real productivity/task data instead of placeholder/hardcoded values, without changing any UI.

## Constraints & Preferences
- *UI must remain identical*: no changes to layout, typography, spacing, colors, card structure, chart structure, or component hierarchy.
- Tasks box: only track `has_deadline=True`, count unique `task_group_id` (never double-count start+deadline records).
- Tasks denominator: status `"In Progress"` OR (event_date < today AND deadline_date > today).
- Tasks numerator: status `"Done"`.
- Productivity box: only track `has_deadline=False`, today only.
- Productivity formula: Productive minutes / (Productive + Unproductive minutes). Ignore Neutral and Obligation completely.
- Weekly Overview: real completed task data grouped by day via `status_change_at`, real productive duration data, dynamic chart scaling to max hours in viewed week.
- Navigation arrows must recalculate all analytics on week change.
- Today highlight in viewed week only (no highlight if today is outside the viewed week).
- Edge case: if productive=0 and unproductive=0, display 0% (no NaN, no divide-by-zero).
